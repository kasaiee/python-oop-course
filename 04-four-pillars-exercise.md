# تمرین‌های فصل چهارم: اصول چهارگانه شیءگرایی

> این تمرین‌ها هر چهار ستون شیءگرایی (ارث‌بری، انتزاع، چندریختی، کپسول‌سازی) را به همان ترتیب فصل پوشش می‌دهند. ابتدا خودتان پاسخ دهید، سپس با `04-four-pillars-answer.md` مقایسه کنید.

---

## بخش الف: ارث‌بری

**۱.** ارث‌بری چه مشکلی را حل می‌کند؟ «آزمون Is-A» چیست؟ سپس برای هر جفت زیر بگویید ارث‌بری درست است یا غلط، و چرا:

- `Manager` از `Employee`
- `Car` از `Engine`
- `Penguin` از `Bird`
- `Order` از `OrderItem`

**۲.** `super()` چه کاری انجام می‌دهد و چرا آن را به صدازدن مستقیم نام والد (مثل `Employee.__init__(self, ...)`) ترجیح می‌دهیم؟ سپس بگویید کد زیر دقیقاً کجا و با چه خطایی می‌شکند:

```python
class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.salary = salary

class BrokenManager(Employee):
    def __init__(self, name, salary, team_size):
        self.team_size = team_size

b = BrokenManager("Sara", 8_000_000, 3)
print(b.name)
```

**۳.** تفاوت «Override کامل» و «Override توسعه‌ای (با `super`)» را با یک مثال کوتاه بیان کنید. قاعده‌ی انتخاب بین این دو چیست؟

**۴.** چهار نوع ارث‌بری (ساده، چندسطحی، سلسله‌مراتبی، چندگانه) را تعریف کنید و برای هرکدام یک مثال از دنیای واقعی بزنید. کدام نوع در پروژه‌های واقعی پرکاربردتر است؟ ریسک اصلی وراثت چندگانه چیست؟

**۵.** چرا برای خطاهای خاص پروژه، کلاس‌های استثنای سفارشی (با ارث‌بری از `Exception`) تعریف می‌کنیم؟ مزیت ساختن یک **سلسله‌مراتب** از استثناها چیست؟

## بخش ب: انتزاع

**۶.** انتزاع را تعریف کنید — با یک مثال غیربرنامه‌نویسی و دو مثال از خود پایتون که هر روز بی‌آنکه بدانید مصرف‌کننده‌ی انتزاع‌شان بوده‌اید.

**۷.** منظور از «لایه‌های انتزاع» چیست و چرا مهم است که در هر لحظه فقط به یک لایه فکر کنیم؟ در زنجیره‌ی `notify_user → sms.send → requests.post` هر لایه چه چیزی را از لایه‌ی بالاتر پنهان می‌کند؟

**۸.** کلاس انتزاعی (`ABC` و `@abstractmethod`) چه چیزی به انتزاع اضافه می‌کند؟ چرا `Shape()` مستقیماً قابل ساختن نیست؟ و اگر زیرکلاسی متد انتزاعی را پیاده نکند، دقیقاً چه اتفاقی و **در چه لحظه‌ای** می‌افتد؟ چرا این «لحظه» مزیت مهمی است؟

**۹.** جمله‌ی «رابط را حول نیاز استفاده‌کننده طراحی کنید، نه حول ساختار درونی» یعنی چه؟ با مقایسه‌ی دو رابط فرضی برای کلاس پیامک توضیح دهید: `send(to, message)` در برابر `set_auth_header(...)` + `post_json(...)`.

## بخش ج: چندریختی

**۱۰.** چندریختی چیست و چه تفاوتی با «Override» دارد؟ (به تفاوت «مکانیزم» و «نتیجه» اشاره کنید.)

**۱۱.** Duck Typing چیست؟ در تابع زیر، چرا `total_area` نیازی ندارد بداند با دایره کار می‌کند یا مستطیل — و این چه بهایی دارد؟

```python
def total_area(shapes):
    return sum(shape.area() for shape in shapes)
```

**۱۲.** سه مسیر رسیدن به چندریختی در پایتون را نام ببرید. سپس برای هر سناریو بگویید کدام مسیر مناسب‌تر است و چرا:

- یک اسکریپت شخصی کوچک برای پردازش چند نوع فایل
- درگاه‌های پرداخت متعدد در محصول یک تیم بیست‌نفره
- سلسله‌مراتب کارمندان که رفتار مشترک قابل‌توجهی هم دارند

**۱۳.** تفاوت Overriding و Overloading چیست؟ پایتون به‌جای Overloading (به سبک جاوا) چه سازوکاری دارد؟ یک مثال کوتاه بزنید.

## بخش د: کپسول‌سازی

**۱۴.** کپسول‌سازی چیست و چه مشکلی را حل می‌کند؟ در کد زیر چه فاجعه‌ای رخ می‌دهد و کپسول‌سازی چطور جلوی آن را می‌گیرد؟ در پاسخ‌تان از مفهوم **ناوردا (Invariant)** استفاده کنید.

```python
acc = BankAccount(1000)
acc.balance = -5000
```

**۱۵.** جدول زیر را درباره‌ی سه سطح دسترسی در پایتون کامل کنید:

| نام‌گذاری | نام سطح | معنا |
|----------|---------|------|
| `name` | ؟ | ؟ |
| `_name` | ؟ | ؟ |
| `__name` | ؟ | ؟ |

**۱۶.** Name Mangling چیست و پایتون `__pin` را در کلاس `Account` به چه چیزی تبدیل می‌کند؟ آیا این یک مکانیزم امنیتی واقعی است؟ سپس با سناریوی `Logger`/`TimedLogger` (هر دو با attribute خصوصی `__count`) توضیح دهید هدف اصلی این سازوکار چیست و بدون آن چه اتفاقی می‌افتاد.

**۱۷.** شعار *«We are all consenting adults here»* در پایتون به چه معناست؟ Tradeoff رویکرد پایتون (قرارداد) در برابر رویکرد جاوا (`private` اجباری) چیست؟

## بخش ه: کدنویسی و کدخوانی

**۱۸. (کدخوانی)** خروجی `C().setup()` در کد زیر چیست و چرا به این ترتیب؟

```python
class A:
    def setup(self):
        print("A")

class B(A):
    def setup(self):
        super().setup()
        print("B")

class C(B):
    def setup(self):
        super().setup()
        print("C")
```

**۱۹. (کپسول‌سازی)** یک کلاس `Temperature` بنویسید که دما را برحسب سلسیوس نگه دارد، اما اجازه ندهد مقداری کمتر از صفر مطلق (`-273.15`) تنظیم شود. برای خواندن و نوشتن کنترل‌شده، از یک attribute داخلی (`_celsius`) و متدهای `get`/`set` استفاده کنید.

**۲۰. (انتزاع + چندریختی)** یک کلاس انتزاعی `Shape` با متد انتزاعی `area()` تعریف کنید. سپس سه شکل (`Circle`، `Rectangle`، `Triangle`) را پیاده کنید و تابع `describe_all(shapes)` را بنویسید که مساحت همه را بدون هیچ `if/elif`ی روی نوع چاپ کند. در پایان توضیح دهید افزودن شکل چهارم چطور انجام می‌شود و چرا `describe_all` تغییر نمی‌کند.

**۲۱. (کدخوانی — کپسول‌سازی)** در کلاس حساب بانکی فصل، متد تاریخچه چنین بود:

```python
def get_history(self):
    return list(self._transactions)
```

چرا به‌جای `return self._transactions` از `list(...)` استفاده شده؟ اگر لیست اصلی مستقیم برگردانده می‌شد، با چه کدی از بیرون می‌شد کپسول را شکست؟
