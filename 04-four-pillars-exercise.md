## بخش الف: کپسول‌سازی

**۱.** کپسول‌سازی چیست و چه مشکلی را حل می‌کند؟ در کد زیر چه فاجعه‌ای رخ می‌دهد و کپسول‌سازی چطور جلوی آن را می‌گیرد؟

```python
acc = BankAccount(1000)
acc.balance = -5000
```

**۲.** جدول زیر را درباره‌ی سه سطح دسترسی در پایتون کامل کنید:

| نام‌گذاری | نام سطح | معنا |
|----------|---------|------|
| `name` | ؟ | ؟ |
| `_name` | ؟ | ؟ |
| `__name` | ؟ | ؟ |

**۳.** Name Mangling چیست؟ در کلاس زیر، پایتون `__pin` را به چه چیزی تبدیل می‌کند؟ آیا این یک مکانیزم امنیتی واقعی است؟ هدف اصلی آن چیست؟

```python
class Account:
    def __init__(self):
        self.__pin = "1234"
```

**۴.** شعار *«We are all consenting adults here»* در پایتون به چه معناست؟ Tradeoff رویکرد پایتون (قرارداد) در برابر رویکرد جاوا (`private` اجباری) چیست؟

## بخش ب: انتزاع و ارث‌بری

**۵.** تفاوت ظریف «انتزاع» و «کپسول‌سازی» را توضیح دهید. هرکدام به کدام سوال پاسخ می‌دهند؟

**۶.** ارث‌بری چه نوع رابطه‌ای می‌سازد و چه مشکلی را حل می‌کند؟ چه زمانی **نباید** از ارث‌بری استفاده کنیم؟

**۷.** `super()` چه کاری انجام می‌دهد؟ تفاوت «Override کامل» و «Override توسعه‌ای (با `super`)» را با یک مثال کوتاه بیان کنید.

**۸.** تفاوت `__str__` و `__repr__` چیست؟ هرکدام برای چه مخاطبی‌اند و کی صدا زده می‌شوند؟ در عبارت `print([m])` (یعنی وقتی شیء داخل یک لیست است) کدام‌یک استفاده می‌شود؟

**۹.** چرا برای خطاهای خاص پروژه، کلاس‌های استثنای سفارشی (با ارث‌بری از `Exception`) تعریف می‌کنیم؟ مزیت ساختن یک سلسله‌مراتب از استثناها چیست؟

## بخش ج: چندریختی

**۱۰.** چندریختی چیست و چه تفاوتی با «Override» دارد؟ (به تفاوت «مکانیزم» و «نتیجه» اشاره کنید.)

**۱۱.** Duck Typing چیست؟ در تابع زیر، چرا `total_area` نیازی ندارد بداند با دایره کار می‌کند یا مستطیل؟

```python
def total_area(shapes):
    return sum(shape.area() for shape in shapes)
```

**۱۲.** تفاوت `isinstance(obj, Cls)` و `type(obj) == Cls` چیست؟ چرا در کد شیءگرا معمولاً `isinstance` انتخاب درست‌تری است؟

## بخش د: کدنویسی و کدخوانی

**۱۳. (کدخوانی)** خروجی `C().setup()` در کد زیر چیست و چرا به این ترتیب؟

```python
class A:
    def setup(self):
        print("A")

class B(A):
    def setup(self):
        super().setup()
        print("B")

class C(B):
    def setup(self):
        super().setup()
        print("C")
```

**۱۴. (کپسول‌سازی)** یک کلاس `Temperature` بنویسید که دما را برحسب سلسیوس نگه دارد، اما اجازه ندهد مقداری کمتر از صفر مطلق (`-273.15`) تنظیم شود. برای خواندن و نوشتن کنترل‌شده، از یک attribute داخلی (`_celsius`) و متدهای `get`/`set` استفاده کنید.

**۱۵. (چندریختی)** ابتدا سه کلاس شکل (`Circle`، `Rectangle`، `Triangle`) بنویسید که هرکدام متد `area()` داشته باشند. سپس یک تابع `describe_all(shapes)` بنویسید که مساحت همه را بدون استفاده از `if/elif` روی نوع، چاپ کند. در پایان توضیح دهید افزودن یک شکل جدید چطور انجام می‌شود.

**۱۶. (متد جادویی `__call__`)** یک کلاس `Adder` بنویسید که با یک عدد پایه ساخته شود و وقتی مثل تابع صدا زده شد (`add(5)`)، آن عدد را به پایه اضافه کند. نشان دهید `callable(instance)` برابر `True` است.
