## بخش الف: مدلِ حافظه‌ی پایه

**۱. (کدخوانی)** خروجیِ کدِ زیر چیست و چرا؟ توضیح دهید چرا `b = a` یک «کپی» نیست.

```python
a = [1, 2, 3]
b = a
b.append(4)
print(a)
print(a is b)
```

**۲. (کدخوانی + رفع اشکال)** خروجیِ کدِ زیر چیست و چرا این یک «دام» است؟ آن را اصلاح کنید تا هر سبد لیستِ مستقلِ خودش را داشته باشد.

```python
class ShoppingCart:
    def __init__(self, items=[]):
        self.items = items

c1 = ShoppingCart()
c2 = ShoppingCart()
c1.items.append("Book")
print(c2.items)
```

**۳.** تفاوتِ «شمارشِ ارجاع» (Reference Counting) و «Garbage Collector» چیست؟ چرا پایتون علاوه بر شمارشِ ارجاع به GC هم نیاز دارد؟ (به مفهومِ «ارجاعِ حلقوی» اشاره کنید.)

## بخش ب: `__del__` و Flyweight

**۴.** متدِ `__del__` چه کاری انجام می‌دهد و چرا تقریباً هیچ‌کس از آن استفاده نمی‌کند؟ جایگزینِ درست برای آزادکردنِ مطمئنِ منابع چیست؟

**۵. (کدخوانی)** خروجیِ کدِ زیر چه می‌تواند باشد و چرا؟ این چه ارتباطی با الگوی Flyweight دارد؟

```python
a = 256
b = 256
print(a is b)

x = 1000
y = 1000
print(x is y)
```

## بخش ج: `__slots__` و `weakref`

**۶.** `__slots__` چیست و چطور مصرفِ حافظه را کاهش می‌دهد؟ چه تأثیری بر `__dict__`ِ شیء دارد؟

**۷.** چه زمانی **نباید** از `__slots__` استفاده کنیم؟ دست‌کم دو معایب/محدودیتِ آن را نام ببرید و Tradeoffِ اصلی را توضیح دهید.

**۸.** یک «ارجاعِ ضعیف» (weak reference) چیست و چه تفاوتی با یک ارجاعِ معمولی (قوی) دارد؟ در کدِ زیر، چرا بعد از `del d`، مقدارِ `weak()` برابرِ `None` می‌شود؟

```python
import weakref
class Data:
    def __init__(self, v): self.value = v

d = Data(42)
weak = weakref.ref(d)
del d
print(weak())
```

**۹.** `weakref` چطور از «نشتِ حافظه» (Memory Leak) جلوگیری می‌کند؟ کاربردِ آن در یک «کش» را توضیح دهید.

**۱۰.** رابطه‌ی `__slots__` و `weakref` چیست؟ چرا وقتی از `__slots__` استفاده می‌کنیم و هم‌زمان weakref می‌خواهیم، باید `'__weakref__'` را به فهرستِ slots اضافه کنیم؟

## بخش د: کپی کردن و کدنویسی

**۱۱. (کدخوانی)** تفاوتِ کپیِ سطحی (shallow) و عمیق (deep) چیست؟ خروجیِ کدِ زیر چیست و چرا؟

```python
import copy
class Order:
    def __init__(self, items): self.items = items

original = Order(["Book", "pen"])
shallow = copy.copy(original)
shallow.items.append("notebook")
print(original.items)
```

**۱۲.** چرا اشیای تغییرناپذیر (Immutable) مثلِ `tuple` و `frozenset` «نیازی به کپی ندارند»؟ این چه مزیتی در برنامه‌های چندنخی دارد؟

**۱۳. (کدنویسی)** یک کلاسِ `Color` با الگوی Flyweight بنویسید که اگر رنگی با نامِ مشخص قبلاً ساخته شده باشد، **همان شیء** را برگرداند نه یک نمونه‌ی تازه. (راهنمایی: از `__new__` و یک کشِ کلاسی استفاده کنید.) نشان دهید `Color("red") is Color("red")` برابرِ `True` است.

**۱۴. (کدنویسی)** یک کلاسِ `Cart` بنویسید که: (الف) دامِ آرگومانِ پیش‌فرضِ تغییرپذیر را نداشته باشد، و (ب) از `__slots__` برای صرفه‌جویی حافظه استفاده کند (فرض کنید فقط یک attribute به نامِ `items` دارد).

## بخش ه: پرسشِ تحلیلی

**۱۵.** نویسنده می‌گوید بیشترِ این ابزارها (`__slots__`، `weakref`، Flyweight) «بهینه‌سازی‌اند و بهینه‌سازیِ زودرس ریشه‌ی خیلی از بدی‌هاست»، اما در همان حال تأکید می‌کند «دام‌های حافظه را از همان اول بشناسید چون آن‌ها باگ‌اند نه بهینه‌سازی». این تمایز بینِ «باگ» و «بهینه‌سازی» را توضیح دهید و برای هرکدام یک مثال از فصل بزنید.

**۱۶.** این جمله را نقد کنید: «چون `__slots__` حافظه را کم می‌کند، بهترین کار این است که در همه‌ی کلاس‌هایمان از آن استفاده کنیم.»

## بخش و: در عمق حافظه

**۱۷. (کدخوانی)** خروجی را پیش‌بینی کنید و توضیح دهید چرا با وجودِ `del` روی هر دو نام، پیش از `gc.collect()` این دو شیء آزاد نشده‌اند:

```python
import gc

class Node:
    pass

a = Node()
b = Node()
a.partner = b
b.partner = a

del a
del b
found = gc.collect()
print(found > 0)
```

**۱۸. (کدنویسی)** یک `SessionCache` با `weakref.WeakValueDictionary` بنویسید: متدِ `put(key, session)` و `get(key)`. نشان دهید تا وقتی جای دیگری ارجاعِ قوی به session هست، از کش خوانده می‌شود، و به‌محضِ `del` شدنِ آن ارجاع، کلید خودکار از کش غیب می‌شود.

**۱۹. (کدخوانی/دام)** والد `__slots__` دارد اما زیرکلاس نه. پیش‌بینی کنید: آیا خطِ آخر خطا می‌دهد؟ صرفه‌جوییِ حافظه‌ی والد برای نمونه‌های زیرکلاس چه می‌شود؟

```python
class Point:
    __slots__ = ("x", "y")
    def __init__(self, x, y):
        self.x, self.y = x, y

class NamedPoint(Point):
    def __init__(self, x, y, name):
        super().__init__(x, y)
        self.name = name

p = NamedPoint(1, 2, "origin")
p.extra = "surprise!"
print(p.extra, p.__dict__)
```

**۲۰. (کدنویسی)** کلاسِ `Document` دو فیلد دارد: `content` (لیستِ پاراگراف‌ها — باید در کپیِ عمیق واقعاً کپی شود) و `settings` (دیکشنریِ پیکربندیِ مشترکِ کلِ اپلیکیشن — **نباید** کپی شود؛ همه‌ی سندها باید همان یک شیء را ببینند). متدِ `__deepcopy__` را طوری پیاده کنید که این سیاست را اجرا کند و با آزمایش نشانش دهید.
