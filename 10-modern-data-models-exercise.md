# تمرین‌های فصل دهم: مدل‌های داده مدرن — dataclass، NamedTuple، Enum و TypedDict

> این تمرین‌ها ابزارهای مدرن ساخت کلاس‌های حامل داده را پوشش می‌دهند. ابتدا خودتان پاسخ دهید، سپس با `10-modern-data-models-answer.md` مقایسه کنید.

## تمرین ۱ - dataclass

**۱.** مشکل نوشتن دستی یک کلاس حامل داده با فیلدهای زیاد چیست؟ `@dataclass` در پشت صحنه کدام متدها را خودکار می‌سازد؟

**۲. (کدنویسی)** یک کلاس `Product` با `@dataclass` بنویسید که فیلدهای `name` (رشته)، `price` (عدد)، `stock` (عدد، پیش‌فرض ۰) و `category` (رشته، پیش‌فرض «عمومی») داشته باشد. سپس یک نمونه بسازید و آن را چاپ کنید.

**۳.** پارامترهای `frozen=True` و `order=True` در `@dataclass` چه می‌کنند؟ چرا `frozen=True` یک مزیت اضافه (ساخت `__hash__`) هم به‌همراه دارد و این مزیت چه امکانی می‌دهد؟

**۴.** متد `__post_init__` چیست و **کی** اجرا می‌شود؟ یک کاربرد رایج آن را نام ببرید.

**۵.** چرا در یک dataclass نمی‌توانید مستقیم `items: list = []` بنویسید؟ راه درست چیست و تفاوت `default` با `default_factory` را توضیح دهید.

## تمرین ۲ - dataclass در برابر Pydantic

**۶. (کدخوانی)** آیا کد زیر خطا می‌دهد؟ خروجی `print` چیست؟ این چه چیزی درباره‌ی اعتبارسنجی نوع در `@dataclass` به ما می‌گوید؟

```python
from dataclasses import dataclass

@dataclass
class Product:
    price: int

p = Product("string!")
print(p.price)
```

**۷.** تفاوت اصلی `@dataclass` و `Pydantic.BaseModel` چیست؟ Tradeoff هرکدام کدام است و برای هر نوع داده (داخلی مطمئن در برابر ورودی نامطمئن) کدام مناسب‌تر است؟

## تمرین ۳ - NamedTuple، TypedDict و Enum

**۸.** `NamedTuple` چیست و چه زمانی مناسب است؟ دو رفتاری که از `tuple` به ارث می‌برد (که `dataclass` ندارد) را نام ببرید.

**۹.** `TypedDict` چیست؟ خروجی نهایی آن چه نوعی است (کلاس یا دیکشنری)؟ چه زمانی به‌جای dataclass از آن استفاده می‌کنیم؟

**۱۰.** `Enum` چیست و چرا برای نگه‌داری حالت‌هایی مثل «وضعیت سفارش» بهتر از یک رشته یا عدد است؟ در کد زیر، مشکل استفاده از رشته چیست؟

```python
if order_status == "penging":   # ?
    ...
```

**۱۱.** دکوراتور `@unique` روی یک Enum چه تضمینی می‌دهد؟

## تمرین ۴ - کدنویسی و انتخاب

**۱۲. (کدنویسی)** یک `Enum` به نام `OrderStatus` با حالت‌های `PENDING`، `PAID`، `SHIPPED` و `DELIVERED` بسازید. سپس یک تابع `can_cancel(status)` بنویسید که فقط برای `PENDING` و `PAID` مقدار `True` برگرداند.

**۱۳. (کدنویسی + مقایسه)** یک نقطه‌ی دوبعدی **تغییرناپذیر** را به دو روش بسازید: (الف) با `@dataclass(frozen=True)`، (ب) با `NamedTuple`. سپس یک تفاوت عملی این دو را بیان کنید.

**۱۴.** جدول زیر را با ابزار مناسب پر کنید (`dataclass`، `frozen dataclass`، `NamedTuple`، `TypedDict`، `Enum` یا `Pydantic`):

| سناریو | ابزار مناسب |
|--------|--------------|
| مجموعه‌ی محدودی از نقش‌های کاربر (admin/editor/viewer) | ؟ |
| داده‌ی ورودی یک API که باید اعتبارسنجی شود | ؟ |
| یک رکورد مختصات `(lat, lng)` که مثل tuple هم unpack شود | ؟ |
| ساختار یک پاسخ JSON که می‌خواهید فقط تایپ‌چک شود | ؟ |

## تمرین ۵ - پرسش تحلیلی

**۱۵.** نویسنده می‌گوید این ابزارها «جایگزین همه‌ی کلاس‌ها نیستند». چه زمانی یک کلاس معمولی بهتر از `dataclass` است؟ دو سوالی که برای انتخاب ابزار درست باید بپرسید («آیا این عمدتاً داده است یا رفتار؟» و «آیا باید تغییرپذیر باشد؟») را توضیح دهید.

**۱۶.** این جمله را نقد کنید: «چون `@dataclass` کد کمتری دارد و تمیزتر است، بهتر است همه‌ی کلاس‌های پروژه را به dataclass تبدیل کنیم.»
