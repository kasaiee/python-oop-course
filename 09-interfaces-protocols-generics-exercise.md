## بخش الف: Abstract Base Classes (ABC)

**۱.** کلاسِ انتزاعی (ABC) چیست و چه مزیتی نسبت به یک کلاسِ معمولی دارد؟ منظور از «شکستِ زودهنگام» (fail early) در ABC چیست؟

**۲. (کدخوانی)** در کدِ زیر، آیا `BrokenGateway()` خطا می‌دهد؟ اگر بله، **کِی** — هنگامِ ساختِ شیء یا هنگامِ فراخوانیِ `refund`؟ چرا؟

```python
from abc import ABC, abstractmethod

class PaymentGateway(ABC):
    @abstractmethod
    def pay(self, amount): ...
    @abstractmethod
    def refund(self, amount): ...

class BrokenGateway(PaymentGateway):
    def pay(self, amount):
        return "..."

BrokenGateway()
```

**۳.** چه زمانی استفاده از ABC مناسب است و چه زمانی نه؟

## بخش ب: Protocolها

**۴.** Protocol چیست و «Structural Subtyping» (زیرنوعیِ ساختاری) یعنی چه؟ چه تفاوتی با روشِ کارِ ABC دارد؟

**۵.** جدولِ زیر را درباره‌ی تفاوتِ ABC و Protocol کامل کنید:

| ویژگی | ABC | Protocol |
|-------|-----|----------|
| نوعِ رابطه | ؟ | ؟ |
| الزامِ ارث‌بری | ؟ | ؟ |
| مناسبِ | ؟ | ؟ |

**۶.** `@runtime_checkable` چه کاری می‌کند؟ Tradeoff یا محدودیتِ مهمِ آن هنگامِ استفاده از `isinstance` روی یک Protocol چیست؟

**۷. (سناریو)** می‌خواهید تابعی بنویسید که با **هر چیزی که متدِ `read()` دارد** کار کند — چه `open()` استانداردِ پایتون، چه یک کلاس از کتابخانه‌ای که کدش را کنترل نمی‌کنید. برای این کار ABC مناسب است یا Protocol؟ چرا؟

## بخش ج: ژنریک‌ها (Generics)

**۸.** مشکلِ نوشتنِ یک «مخزن» (Repository) بدونِ ژنریک چیست؟ چرا استفاده از `Any` برای این کار، ایمنیِ نوع را از بین می‌برد؟

**۹. (کدنویسی)** یک کلاسِ `Repository` ژنریک بنویسید (با `Generic[T]`) که متدهای `add(item)` و `get(index)` داشته باشد. سپس نشان دهید چطور می‌توان `Repository[User]` و `Repository[Product]` ساخت.

**۱۰.** `TypeVar` با `bound` چیست و چه مشکلی را حل می‌کند؟ در مثالِ `T = TypeVar("T", bound=Comparable)`، اگر `bound` را برنداریم چه اتفاقی می‌افتد؟

## بخش د: تایپ‌هینترهای پیشرفته و کدخوانی

**۱۱.** تفاوتِ `Optional[str]` و `str | None` چیست؟ آیا `Optional[str]` به‌معنای «آرگومانِ اختیاری» است؟ توضیح دهید.

**۱۲.** خطرِ استفاده از `Any` چیست؟ در کدِ زیر چرا تایپ‌چکر هیچ هشداری نمی‌دهد؟ یک جایگزینِ امن‌تر برای `Any` چه می‌تواند باشد؟

```python
from typing import Any

def process(data: Any):
    return data.anything()
```

**۱۳.** `@overload` چه کاربردی دارد؟ در زمانِ اجرا (runtime) کدام تعریف واقعاً اجرا می‌شود؟

**۱۴.** چرا استفاده از `Iterable[int]` به‌عنوانِ نوعِ پارامترِ یک تابع، اغلب بهتر از `list[int]` است؟ این با کدام اصل از فصل هشتم هم‌راستاست؟

## بخش ه: پرسشِ تحلیلی

**۱۵.** نویسنده می‌گوید: «بین ABC و Protocol، اگر شک داری، Protocol را انتخاب کن.» چرا این توصیه با فلسفه‌ی پایتون سازگارتر است؟
