# تمرین‌های فصل یازدهم: استثناها، سریال‌سازی و Introspection

## بخش الف: استثناها

**۱.** چرا در یک پروژه‌ی واقعی به استثناهای سفارشی نیاز داریم؟ استفاده از یک `ValueError` عمومی برای همه‌ی خطاها چه اطلاعاتی را از دست می‌دهد؟ یک استثنای سفارشی چطور می‌تواند «داده‌ی اضافی» حمل کند؟

**۲. (کدنویسی)** یک سلسله‌مراتب از استثناها بسازید: `ShopError` به‌عنوان پایه، `PaymentError` و `InventoryError` مشتق از آن، و `OutOfStockError` مشتق از `InventoryError`. سپس نشان دهید چطور می‌توان یک `OutOfStockError` را هم در سطح `InventoryError` و هم در سطح `ShopError` گرفت.

**۳.** عبارت `raise NewError(...) from original` چه کاری انجام می‌دهد؟ `__cause__` چیست و چه مزیتی در دیباگ دارد؟

**۴.** `raise ... from None` چه اثری دارد؟ چه زمانی مفید است و Tradeoff آن چیست؟

## بخش ب: سریال‌سازی

**۵.** سریال‌سازی (Serialization) چیست؟ دو تفاوت مهم `json` و `pickle` را بیان کنید.

**۶. (کدنویسی)** یک کلاس `User` (با `name` و `age`) بنویسید که متد `to_dict()` و یک `@classmethod` به نام `from_dict()` داشته باشد، سپس نشان دهید چطور با `json.dumps`/`json.loads` آن را ذخیره و بازسازی می‌کنید.

**۷.** متدهای `__getstate__` و `__setstate__` چه کاری انجام می‌دهند؟ در کد زیر، چرا `socket` از حالت ذخیره‌شده حذف و هنگام بازسازی از نو ساخته می‌شود؟

```python
class Connection:
    def __init__(self, host):
        self.host = host
        self.socket = "live connection"

    def __getstate__(self):
        state = self.__dict__.copy()
        del state["socket"]
        return state

    def __setstate__(self, state):
        self.__dict__.update(state)
        self.socket = "new connection"
```

**۸.** چرا `pickle` «امنیت ندارد»؟ قاعده‌ی امنیتی مطلق فصل درباره‌ی آن چیست و چه جایگزین‌های امنی وجود دارد؟

## بخش ج: Introspection

**۹.** تفاوت `obj.__dict__`، `dir(obj)` و `vars(obj)` چیست؟ هرکدام چه چیزی نشان می‌دهند؟

**۱۰.** توابع `hasattr`، `getattr` و `setattr` چه کاری می‌کنند؟ کاربرد اصلی آن‌ها (کار با attribute که نامش در زمان نوشتن کد معلوم نیست) را توضیح دهید. در `getattr(p, "price", 0)`، نقش آرگومان سوم (`0`) چیست؟

**۱۱.** `__annotations__` چیست و چه چیزی در آن ذخیره می‌شود؟ چه ارتباطی با کارکرد `dataclass` و `Pydantic` دارد؟

**۱۲.** ماژول `inspect` چه امکاناتی می‌دهد؟ دو کاربرد آن (مثلاً گرفتن فهرست متدها یا امضای یک تابع) را نام ببرید.

## بخش د: کدنویسی و کدخوانی

**۱۳. (کدخوانی)** برای کلاس زیر، خروجی `p.__dict__` و `vars(p)` چیست؟ آیا `category` در `p.__dict__` هست؟ چرا؟

```python
class Product:
    category = "general"
    def __init__(self, name):
        self.name = name

p = Product("Keyboard")
```

**۱۴. (کدنویسی)** با استفاده از `@contextmanager` از ماژول `contextlib`، یک context manager به نام `timer` بسازید که مدت‌زمان اجرای بلوک `with` را چاپ کند. (این را بدون نوشتن کلاس کامل `__enter__`/`__exit__` انجام دهید.)

## بخش ه: پرسش تحلیلی

**۱۵.** نویسنده می‌گوید: «وسوسه‌ی استفاده‌ی افراطی از Introspection (کد بیش‌ازحد پویا) را مهار کنید؛ کد صریح تقریباً همیشه خواناتر است.» با یک مثال توضیح دهید که چطور استفاده‌ی بیش‌ازحد از `getattr`/`setattr` می‌تواند کد را سخت‌فهم کند و چه زمانی استفاده از آن واقعاً موجه است.

**۱۶.** چرا ساختن یک سلسله‌مراتب تمیز استثنا (یک پایه برای کل پروژه) یک «سرمایه‌گذاری کوچک» است که در کل عمر پروژه سود می‌دهد؟ این چه نفعی برای کسی که از کتابخانه‌ی شما استفاده می‌کند دارد؟
