## تمرین ۱ - وراثت چندگانه و مسئله‌ی الماس

**۱.** چرا پایتون وراثت چندگانه را اجازه می‌دهد اما جاوا آن را ممنوع کرده است؟ رویکرد هر زبان (پایتون: «اجازه بده اما با قاعده» / جاوا: «ممنوع کن تا امن بماند») را با فلسفه‌ی کلی‌شان مرتبط کنید.

**۲.** مسئله‌ی الماس (Diamond Problem) چیست؟ آن را با یک نمودار توضیح دهید و بگویید چه ابهامی ایجاد می‌کند.

**۳. (کدخوانی)** خروجی `D()` در کد زیر چیست؟ چرا کلاس `A` با اینکه هم `B` و هم `C` از آن ارث می‌برند، فقط **یک‌بار** اجرا می‌شود؟

```python
class A:
    def __init__(self):
        print("A")

class B(A):
    def __init__(self):
        print("B")
        super().__init__()

class C(A):
    def __init__(self):
        print("C")
        super().__init__()

class D(B, C):
    def __init__(self):
        print("D")
        super().__init__()

D()
```

## تمرین ۲ - MRO و الگوریتم C3

**۴.** MRO مخفف چیست و چه کاری می‌کند؟ سه قاعده‌ای که الگوریتم C3 تضمین می‌کند را نام ببرید.

**۵. (MRO دستی)** برای سلسله‌مراتب زیر، MRO را **روی کاغذ** بسازید (بدون اجرا)، سپس بگویید `D.mro()` چه فهرستی برمی‌گرداند:

```python
class A: pass
class B(A): pass
class C(A): pass
class D(B, C): pass
```

**۶.** جمله‌ی «`super()` نفر بعدی در MRO را صدا می‌زند، نه لزوماً والد مستقیم را» یعنی چه؟ در الماس سوال ۳، وقتی داخل `B` عبارت `super().__init__()` اجرا می‌شود، به کدام کلاس می‌رود و چرا این «عجیب» به‌نظر می‌رسد؟

**۷.** چرا در وراثت چندگانه باید همیشه از `super()` استفاده کنیم و نه از نام مستقیم والد (مثل `A.__init__(self)`)؟ استفاده‌ی مستقیم چه خطری دارد؟

## تمرین ۴ - مفاهیم پیشرفته و کدنویسی

**۱۱.** متد `__init_subclass__` چیست؟ **کی** صدا زده می‌شود — هنگام تعریف زیرکلاس یا هنگام ساخت شیء؟ چه مزیتی نسبت به متاکلاس دارد؟

**۱۲. (کدنویسی)** یک Mixin به نام `ReprMixin` بنویسید که یک `__repr__` عمومی فراهم کند و نام کلاس به‌همراه محتوای `__dict__` را نشان دهد. سپس آن را به یک کلاس `Point` بچسبانید و نتیجه را نشان دهید.

**۱۳. (کدنویسی)** با استفاده از `__init_subclass__`، یک کلاس پایه‌ی `Plugin` بسازید که هر زیرکلاسش را **به‌طور خودکار** در یک فهرست مشترک (`registry`) ثبت کند. سپس دو زیرکلاس تعریف کنید و نشان دهید هر دو ثبت شده‌اند.

**۱۴.** الگوی کلاس داخلی `Meta` (مثل آنچه در جنگو می‌بینید) چه مزیتی دارد؟ در کد زیر، `User._table` و `Product._table` چه مقداری خواهند داشت؟

```python
class Model:
    def __init_subclass__(cls, **kwargs):
        super().__init_subclass__(**kwargs)
        meta = getattr(cls, "Meta", None)
        cls._table = getattr(meta, "table_name", cls.__name__.lower())

class User(Model):
    class Meta:
        table_name = "users"

class Product(Model):
    pass
```
